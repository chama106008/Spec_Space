# Spec_Space

## Overview

- **Genre**: 2D Platformer  
- **Engine**: Unity 6.0  
- **Concept**: ゲームの「バグ」を乗りこなすアクションゲーム  

---

## Game Design

### ゲームの仕様

基本操作は横方向移動 + ジャンプのシンプルなプラットフォーマー。  
ただしプレイヤーは、自機の以下のパラメータを操作できる：

- 当たり判定
- 重さ
- 物理抵抗

いわばゲームにおける“当たり前の挙動”そのものを調整できる。

本来であれば不利益を生む「不具合」。  
しかし、うまく活用すれば強力な挙動へと変貌する“裏技”となる。

それらを最大限に利用してステージを攻略する、  
**パズル要素を含んだアクションゲーム**である。

---

## Design Philosophy

ゲームの不具合や、それを利用したいわゆる「バグ技」は、  
プログラマー視点では望ましくない挙動である一方で、  
プレイヤー視点ではこの上なく愉快な仕様として好まれる傾向がある。

これは、ゲームが「面白い」と認識される根幹にある  
**“制御感（Sense of Control）”** を強く刺激するためと考えられる。

タスクには条件があり、目標があり、  
そのために使用できるリソースが存在する。

限られた条件の中でリソースを活用し、  
より高い目標を達成した瞬間、  
プレイヤーの脳はゲームを「面白い」と認識する。

バグは本来「条件外」で目標を達成するための手段である。  
その制御感や背徳感、支配感は非常に強い中毒性を持つ。

一方で、プログラマーにとってバグは「不具合」であり、  
想定外の挙動であり、ゲーム破綻の原因にもなりうる恐ろしい存在である。

であるならば——

**いっそのことバグを前提とし、  
不具合ではなく“仕様”として組み込むことで、  
設計者はバグを恐れず、  
プレイヤーはその背徳的な楽しみを享受できるゲームがあっても良いのではないか？**
